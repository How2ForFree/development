-- ORGANIZATION -- First - a note on the organization of this template. I have placed all essential non-html files inside of the "assets" folder. There you'll find the primary images, javascript, css, and other files that are essential to each version of this template.

Above this folder I have grouped each "color" inside of it's own folder. You'll notice that there are 12 main .html files. You will also notice that each of these files is referencing it's respecitive "skin" files at the very top of the header - ie: the white version references skins/quick-styles.css. All skin files are located inside this folder.

The files inside of each theme folder are just for quick access - you can re-organize their proximity to the files in the "assets" folder at your own pleasure.

The CSS file in those folder (quick-styles.css) doesn't cover the main styling for the template, only those styles that differentiate each theme from each other.

A) HTML Structure This theme includes two major layout options: a variable grid layout (the homepage), and a fixed width layout with two possible configurations: Full Width, and 2 Column.
All of the information within the main content area is nested within a div with an id of "grid-content". The header section is located under the "top-bar" ID.

The navigation is located in the "navigation"" class inside the "top-bar".
Each "module" on the homepage is nested inside a DIV with a class of both "post_float" and "rounded".
The main content columns on the other pages are nested inside the div with the ID of "container".
The left column content is within a div with an id of "single_post".
The sidebar's (column #2) content is within a div with an id of "sidebar".
The footer content is within a div with an id of "footer".

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

#content a {<br /> color: #someColor;<br /> }
If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.
.green a {<br /> color: #someColor;<br /> }
So, to ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

The file "style.css" loads all of the major stylesheets for this template. The styling for this theme are stored in 3 key locations: core.css (main styling) , quick-styles.css , and typography.css Finally, there are a handful of other CSS files that include styling for the various third party plugins, such as the dropdown, prettyPhoto, etc. I have taken the time to separate these other stylesheets into the "import" folder.
B) CSS I'm using one main CSS file in this theme to load all other stylesheets (style.css). There are several support stylesheets (for IE browsers and the jQuery stuff) located inside the "import" folder. The site also includes a generic reset stylesheet (reset.css) inside the hacks folder. Many browser interpret the default behavior of html elements differently. By using a general reset CSS rule, we can work round this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

The main file (core.css) contains all of the universal stylings for the pages. The file is separated into sections using:

/* Main Section */
some code

/* Footer */
some code
etc, etc.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

I've also done by best to keep this code as organized as possible. Take a few minutes to familiarize yourself with what's in each file before you start your styling, and it should be a breeze.

G) Customization and Contact Form Usage You can quickly create your own custom stylesheets by over-writing the quick-styles.css file with your own adjustments. Keep in mind that you will need to keep track of the original styling in case your new styles compete with them. You can also add new background graphics and other customizations here sa well.

The Contact Form is quite simple to use, but as it's the only PHP file, I should walk you through it. It's been built using the Super Simple Contact Form Maker : http://supersimple.org/contact. First, there's a good chance that you'll want to ignore the large block of PHP at the top of this page. If all you want to do is setup your email address, just scroll down to line 272-275 in the code:
//now get the recipient(s) $_to[] = "YourEmail@Address.here";
Just enter your email address instead of "YourEmail@Address.here" and the form will be setup. To make customizations to this form, I highly recommend reading the documentation at the Super Simple Contact Form website, as that's what I used to create the form. Just insert the code that they give you into this framework and you'll be good to go.
H) Using the VGrid on the Homepage and Launching the Lightbox This is actually pretty simple, as the plugin does most of the work for you. Keep in mind that all content goes inside the "grid-content" DIV, and tha each module is wrapped with a DIV that has the "post_float" and "rounded" classes. Here's a quick example of what a module looks like, but you'll also notice that I've left additional notes inside the index.html file for you :) <div class="post_float rounded"> <a class="ceebox" title="Lightbox Example" href="destination_path.jpg"> <img width="200" height="130" src="imagepath.jpg" alt="" title="imagesample" /> </a> <h2><a class="ceebox" title="Lightbox Example" href="destination_path.jpg">Lightbox Example</a></h2> <div class="post_tags">This is a Quick Project Description<br /></div> </div> Notice the three main sections of that module - 1) the hyper-linked image thumbnail. 2) the Title wrapped in H2 tags, and 3) the short description wrapped in the "post_tags" DIV.

Launching the Lightbox

For any link that you want to include the lightbox, just include the class of "ceebox" inside the A tag: <a class="ceebox" title="Lightbox Example" href="destination_path.jpg"> <img width="200" height="130" src="imagepath.jpg" alt="" title="imagesample" /> </a> Galleries are handled a little differently because it forces us to load the CEEBOX class outside of the actual A tag (in this example it's loaded in the "post_float" DIV): <div class="post_float rounded ceebox"> <a title="Gallery Example" href="gallery_imagepath1.jpg"> <img width="200" height="130" src="imagepath.jpg" alt="" title="imagesample" /> </a> <div class="gallery-hider"> <a href="gallery_imagepath2.jpg" title="Image Title">link</a> <a href="gallery_imagepath3.jpg" title="Image Title">link</a> </div> ... </div> Note that galleries only work if the 2nd, 3rd, 4th, etc. images are embedded inside a DIV that has a CEEBOX class - in this case we are "hiding" the 2nd, 3rd, 4th, etc. images inside an invisible DIV called "gallery-hider" You can read more about how to use this both inside my HTML comments, and at the official site for the lightbox: CEEBOX OFFICIAL SITE. Please take some time to check out the huge amount of options that the lightbox has - there's a very good chance that any question that you might have for me gets covered somewhere on that page.
